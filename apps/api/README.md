# API Foundation

This FastAPI app owns the dev RAG flow: document replacement, embedded Chroma storage, retrieval, prompt construction, and external Ollama-compatible generation.

## Included

- application entrypoint
- settings
- health route
- placeholder voice endpoint
- document replacement, indexing, retrieval-backed query, and Ollama-compatible model endpoints
- typed schemas
- provider interface stubs
- tests for startup, health, auth, document replacement, and retrieval

## Not Included

- speech pipelines
- background jobs

## API Endpoints

### `PUT /documents` - Replace company document

Replaces the document used by the API with a newly uploaded file. The new document is parsed, chunked, embedded, stored in Chroma, and then stored as the single active company document. Invalid replacements do not overwrite the existing document or active index.

**Authentication:** Bearer token with `Admin` role required.
- No token → `401 Unauthorized`
- Valid token but non-admin role → `403 Forbidden`

**Accepted formats:** `.pdf`, `.docx`, `.txt`

**Constraints:** Maximum file size 10 MB; file must not be empty.

**Success response `200`:**
```json
{ "accepted": true, "filename": "handbook.pdf", "message": "Document 'handbook.pdf' replaced successfully" }
```

**Error responses:** `401`, `403`, `422` (unsupported format / empty file / oversized file)

### `POST /query` - Query indexed company document

Retrieves the highest-ranked chunks from the active company document, builds a company-overview RAG prompt, and returns the generated customer-facing answer. The prompt treats retrieved text as internal company knowledge, not as user-visible sources.

The endpoint accepts recent chat history from the client for follow-up context. The API does not store conversations server-side.

**Request:**
```json
{
  "query": "What does the company do?",
  "top_k": 5,
  "history": [
    { "role": "user", "content": "What services do you offer?" },
    { "role": "assistant", "content": "..." }
  ]
}
```

**Success response `200`:**
```json
{ "answer": "...", "sources": ["company.txt#0"] }
```

The `sources` field is retained for API consumers and debugging, but the website UI should not show source identifiers to customers.

**Validation:** `query` is limited to 1,000 characters, `history` to 12 messages, and each history message to 2,000 characters.

**Error responses:** `400` (no loaded document / empty query), `422` (invalid request shape or oversized input), `502` (LLM provider failure)

## Vector Store

The API uses Chroma through a local `PersistentClient`. Configure the database path and collection name with:

```bash
CHROMA_DB_PATH=data/chroma
CHROMA_COLLECTION_NAME=company-documents
```

Embeddings are generated before vectors are written to Chroma. The API uses the local in-process sentence-transformers provider:

```bash
EMBEDDING_MODEL_NAME=nomic-ai/nomic-embed-text-v1.5
EMBEDDING_TRUST_REMOTE_CODE=true
TRUSTED_REMOTE_EMBEDDING_MODELS=nomic-ai/nomic-embed-text-v1.5
```

Remote model code execution is disabled by default in application settings. Only enable
`EMBEDDING_TRUST_REMOTE_CODE` for model IDs listed in `TRUSTED_REMOTE_EMBEDDING_MODELS`.

After changing `EMBEDDING_MODEL_NAME`, re-upload or reindex documents because existing Chroma vectors were generated by the previous model.

## LLM Provider

Configure the external generate endpoint and chat model with:

```bash
OLLAMA_URL=http://host.docker.internal:11434/api/generate
MODEL_NAME=llama3.1
```

The app does not run a Chroma or Ollama service in Docker.

## Docker Dev

From the repository root:

```bash
docker compose up --build
```

The API is exposed at `http://localhost:8000`, and `./data` is mounted into the container so `data/documents` and `data/chroma` persist across restarts.
