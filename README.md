# Agile Assignment Foundation

This repository is a monorepo foundation for a ChatGPT-style frontend and a FastAPI backend that can later grow into a system similar to [`agile-rag`](https://github.com/MahmoudHilani/agile-rag).

## Structure

```text
.agents/  Project-local agent skills, references, and personas
apps/
  api/   FastAPI backend foundation
  web/   Raw Next.js app generated by create-next-app
docs/
```

## Agent Skills

This repo now vendors the `addyosmani/agent-skills` collection under `.agents/` for project-local use.

- `.agents/skills/`: imported skill folders and `SKILL.md` files
- `.agents/references/`: shared reference checklists from upstream
- `.agents/agents/`: upstream specialist persona files

## Frontend

The frontend was bootstrapped with the official Next.js generator and intentionally left as generated output. No product UI or feature work has been added in this pass.

Install and run it with:

```bash
pnpm --dir apps/web install
pnpm --dir apps/web dev
```

## Backend

The backend is a FastAPI skeleton with explicit extension points for:

- document ingestion
- embeddings
- vector storage
- chat generation
- speech-to-text
- text-to-speech

Nothing above is implemented yet. The scaffold only provides the app entrypoint, configuration, typed schemas, placeholder routes, and tests.

Set up and run it with:

```bash
python -m uv sync --project apps/api
python -m uv run --project apps/api uvicorn app.main:app --reload
```

Run tests with:

```bash
python -m uv run --project apps/api pytest
```

### Admin Auth Note

Admin authentication currently uses an env-configured bcrypt password hash and a bearer JWT stored by the web client in `localStorage`. That is acceptable for this assignment prototype, but it is not a production session model because client-accessible storage can expose the token if the frontend has an XSS issue. Replace it with an httpOnly, secure, sameSite cookie-backed session before production use.
